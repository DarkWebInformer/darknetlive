Phishing: Ex-mod Explains How to Bypass Empire Market's 2FA System

~11 min read | Published on 2019-06-17, tagged Darkweb-Market, Phishing using 2503 words.
In a public paste, one of the most prolific entities responsible for phishing Empire Market users pointed out how easily he—or anyone with the ability to run a proxy—could bypass the marketplace’s “anti-phishing features.” The former Empire Market mod demonstrated how even when presented with the correct Empire Market address, the end user ignores the legitimate Empire Market link in the 2FA message.
“The end user is always the weakest link to a system,” the entity behind many of the Empire Market phishing proxies told darknetlive in an encrypted message. Just days ago, Empire Market staffer Se7en submitted a post on the Empire subdread titled “Market Update (New Anti-Phishing Feature / Disputes).”
Related Post: A Warning About Darkweb Market Phishing Sites
The update, according to the marketplace’s head moderator, added Empire Market addresses to the decrypted PGP messages. This allows users with 2FA enabled to double check the link in the message with the link in the address bar. If the links match, the user is accessing the market through a valid market link. If they do not match, “you are using […] a phishing proxy,” Se7en wrote in the announcement.
In a public paste on the onion service DeepPaste, godman666—an identity used by the Empire Market phisher mentioned in this article—demonstrated that even when confronted with the safe Empire Market link, the end user will often ignore it. And godman666 has some experience with Empire Market fraud; he formerly worked as an Empire Market mod responsible for handling fraud disputes and dealing with fraud on the marketplace’s forum.
The 2FA message contains the code used to login to the marketplace. Since many users just want to access the market before the link stops working, they simply grab the code and close the window with the decrypted 2FA message.

Empire Market 2FA



The proxy passes the encrypted message to the end user; the end user decrypts the message; the end user then unknowingly enters the 2FA code back to the proxy; the proxy passes the 2FA code to Empire Market. This anti-phishing measure would work if the end user had paid any attention to the measure itself.
Below is the message godman666 posted on DeepPaste with the passwords removed. They have already been used most likely. To the users in the paste: this is not Empire Market’s fault. It is yours. The market provided you with a way to prevent this from happening and yet you let it happen. Of course, Empire’s mirrors change by the minute, confusing even responsible users.
Some users, such as those with the username “heythere” and the password “FuckYouPhishingShit” seem to have been paying attention to the link in the 2FA message, proving that not everyone is willing to get phished.
